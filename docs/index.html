deregister(commands, [callback])

Deregisters a key command listener, given matching commands and callback parameters.

parameter type description
commands string or Array<string>

Human readable key or key combination to listen for, in the form of "a" or "shift a" or "control alt a". If multiple keys should perform the same action, pass in an array of strings, e.g. ['a', 'b', 'meta d']
callback [function]

Callback function to execute when key is pressed. This is if you want to only remove key handlers that match the same callback. If you do not provide a callback, all handlers that match commands are removed.

Examples

Keypress.deregister('shift d') // Deregisters all triggers for `shift d`
Keypress.deregister('esc', hide) // Deregisters triggers matching callback function `hide` and key `esc`

init

Initiates keypress manager. Sets a global event listener on the window. This should only have to be called once, when the application bootstraps.

Keypress

Keypress (singleton)

Handles registration, removal, and processing of keyboard inputs. Some inspiration:

  • Keypress.js (http://dmauro.github.io/Keypress/)

register(commands, [options], [callback])

Registers a key command listener with the Keypress Manager

parameter type description
commands string or Array<string>

Human readable key or key combination to listen for, in the form of "a" or "shift a" or "control alt a". If multiple keys should perform the same action, pass in an array of strings, e.g. ['a', 'b', 'meta d']
options [object]

Options that fine tune the behavior of the keypress. If you are unhappy with a default setting, they can be overridden here. Note that it is possible for overrides to conflict with other parameters, for instance, if you set commands to "shift p" but options.shiftKey to false. Don't do this. This is confusing and I won't guarantee preserving how this conflict is addressed.
options.shiftKey [boolean or string] (default false)

If true, the Shift key should be pressed in a key combination. This is automatically set to true if shift is specified in the commands parameter. The boolean test is strict. The string 'optional' can be passed so that the key will fire regardless of whether Shift is pressed.
options.metaKey [boolean or string] (default false)

If true, the "meta" key (Command or Control, depending on the OS) should be pressed in a key combination. This is automatically set to true if meta, control, or command is specified in the commands parameter. The boolean test is strict. The string 'optional' can be passed so that the key will fire regardless of whether Command or Control is pressed.
options.altKey [boolean or string] (default false)

If true, the Alt key should be pressed in a key combination. This is automatically set to true if alt is specified in the commands parameter. The boolean test is strict. The string 'optional' can be passed so that the key will fire regardless of whether Alt is pressed.
options.preventDefault [boolean] (default true)

If true, event.preventDefault() should be called to prevent the key's default behavior.
options.stopPropagation [boolean] (default false)

If true, event.stopPropagation() should be called to prevent other key handlers from triggering. Defaults to false, but as a special case, this is automatically set to true if the command is esc.
options.requireFocusOnBody [boolean] (default true)

If true, the key handler is not triggered if the browser has focused on a specific element, like an input field. Defaults to true, but as a special case, this is automatically set to false if the command is esc.
options.trackMsg [string] (default null)

If a string is provided, the action is logged in event tracking when triggered.
options.trackValue [number] (default null)

If set, and an event is logged, this fills in the value property of a tracked event in Google Analytics.
options.trackOnce [boolean] (default true)

If true, and an event is logged, further identical actions will not be logged.
options.onKeyPress [function]

It is possible to set the callback function to execute on key press on the options object instead of in the callback parameter.
options.condition [function]

If additional conditions are required before executing a callback function after a key is pressed, this function is evaluated. It must return a true or truthy value for the callback to execute.
callback [function]

Function to execute when key is pressed. Technically, this is optional (and you might prefer to set it on options instead). If there is no callback function the keypress simply does nothing.

Examples

Keypress.register('shift d', { trackMsg: 'Shift-D is pressed' }, function () { console.log('Shift-D is pressed!') })
Keypress.register('esc', hide)
